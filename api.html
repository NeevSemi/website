A taste of what's coming

These are representative endpoints from our API surface. Full documentation — parameters, auth, error codes, rate limits — is coming soon.

POST /v1/chat/completions

Route a completion request through LLMGateway. Automatically selects the best available model based on task type, cost envelope, and latency budget.

POST /v1/benchmarks/run

Trigger an LLMScoreBench evaluation run against a specified model and task suite. Returns a run ID for polling results.

GET /v1/benchmarks/{run_id}/results

Retrieve structured benchmark scores for a completed run — per-task, per-dimension, and aggregate quality ratings.

GET /v1/gateway/audit-logs

Fetch paginated semantic audit logs from LLMGateway — prompt hashes, model used, latency, token counts, and compliance flags.

Example request

curl -X POST https://your-gateway.internal/v1/chat/completions \
  -H "Authorization: Bearer <API_KEY>" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "auto",
    "task_hint": "rtl_generation",
    "messages": [{"role": "user", "content": "Generate a 4-bit ripple carry adder in SystemVerilog."}],
    "max_tokens": 1024
  }'
